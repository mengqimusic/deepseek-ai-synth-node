# AI 合成器 Node — 系统设计文档

## 1. 项目定位

一个可在多个正交方向上注入能量的单/复音 AI 合成器节点。通过显式的能量注入操作（按键 / patch point），演奏者培育 5 个独立 Voice 在短期、中期、长期三个时间尺度上发育演化。Voice 之间以频谱资源竞争为主、能量串扰与共鸣耦合为辅，构成一个有内部生态的动态声音系统。

**核心隐喻**：这不是一台"调音色的合成器"，而是一个"可培育的声音生命体"。演奏者的角色从参数调节者转变为能量培育者。

---

## 2. 核心设计决策记录

### 2.1 为什么是显式方向而非隐式？

**决策**：能量方向为显式操作——演奏者有专用的按键 / patch point 来注入特定类型的能量。

**方案 B（选中的）**：传统合成器参数（滤波、FM 指数等）仍保留，但每个参数不直接对应固定声音结果——它设定系统的"当前倾向"，实际输出取决于系统积累的历史。同一旋钮位置，今天和昨天的声音不同——变化来自系统状态，不是随机。

**淘汰的方案 A**：将能量方向编码为演奏动作的隐含维度——系统从正常演奏 gesture 中自动提取并路由能量。淘汰原因：隐式方案下演奏者无法有意识地选择"往哪个方向培育"，发育方向变成系统的自主行为而非演奏者的表达意图。

**讨论要点**：
- 显式方向满足了可演奏控件的六约束，尤其是 Distinct（每个方向彼此明显不同）和 Combinable（组合注入产生涌现交互）
- "确定性"的重新定义：同输入 + 同系统状态 → 同响应。变化来自系统状态的历史积累，不是随机扰动。这保留了确定性，同时获得了演化能力

### 2.2 为什么方向语义选"能量类型"而非"生长维度"？

**决策**：方向定义为能量类型——演奏者投注不同性质的"能量"（如"张""扰""密""忆"），生长维度（系统往哪个方向发育）是能量注入后的涌现结果，非直接操作对象。

**淘汰的方案**：方向直接描述系统的变化轴（如"往谐波密度方向长""往非线性耦合方向长"）。淘汰原因：演奏者的瞬时感知和长期效果脱节——"往谐波密度方向长"按下去的瞬间，演奏者听不到任何东西，控制感丢失。

**讨论要点**：
- 能量类型方案让瞬时反馈和长期发育共享同一个因果链：注入某种能量 → 即刻听到声音变化（感知签名）→ 反复注入 → 系统结构在对应维度上累积偏移 → 长期发育
- 演奏者的心理模型是"我往系统里投注了某种东西"，不是"我设定了一个发育目标"

### 2.3 为什么能量注入的物理对应选"拓扑注入"？

**决策**：能量注入改变系统内部的连接路径——激活新路径、强化已有路径、抑制冗余路径。这是三种物理对应中最适合长线条发育的实现。

**对比方案 A（驱动项注入）**：能量作为外部驱动力加入动力学方程，改变瞬时状态。短期反馈强，但无法累积——每次注入结束后系统回到原状。

**对比方案 B（参数梯度注入）**：能量改变系统内部参数的方向性偏移。能累积，但改变的仍是参数值，不改变系统结构本身——是"调偏置"而非"改拓扑"。

**讨论要点**：
- 拓扑注入天然适合神经网络实现——权重和连接即拓扑
- 短期反馈多样（不同能量激活不同路径类型），长期改变结构（路径强化/抑制形成新的默认状态）
- 与方向语义（能量类型）一致：不同类型的能量激活不同类型的连接路径

### 2.4 为什么选方案 B 能量方向（张/扰/密/忆）？

**决策**：四能量方向——张（Tension）、扰（Turbulence）、密（Density）、忆（Memory）。

**淘汰的方案 A（激/润/裂）**：覆盖频域/时域/结构三维，短期感知签名清晰，但长期发育因果链断裂——"润"延长衰减后，系统结构为什么改变？方案 A 描述的是声音结果，而非系统的变化方向。

**争论要点**：
- 方案 B 原始描述（"有序 ↔ 混沌"类）的致命问题：类别名而非感知签名。"扰"按下后到底听到什么？自由度太大等于随机，违反控制感
- 解决方案：每个方向锁定到具体的、可被演奏者身体感知的机制——

| 方向 | 不是这个 | 是这个（精确感知签名） |
|------|---------|---------------------|
| **张 (Tension)** | "往有序走" | **谐波锁紧** — 相邻分音频率比往整数靠拢，声音从"散"变"实" |
| **扰 (Turbulence)** | "往混沌走" | **边带分叉** — 每个谐波峰旁分裂出一对非谐波小峰，声音从"平滑"中"裂出纹理" |
| **密 (Density)** | "往复杂走" | **路径增生** — 低频段激活更多平行共振路径，声音从"薄片"变"实心" |
| **忆 (Memory)** | "往历史走" | **状态回声** — 过去被强激活过的连接模式以低音量混入当前信号 |

- 长期发育因果链通了：反复注入"扰"→ 边带分叉路径被强化 → 系统默认状态中边带更持久 → 相变后不按"扰"声音也自带纹理

### 2.5 为什么 Neural 直接生成音频，传统合成为 fallback？

**决策**：神经网络直接生成音频波形，若声音质量不达标，传统合成结构作为 fallback。

**理由**：
- 拓扑注入要求系统内部有可被长期改变的结构——神经网络权重天然是这种结构
- 直接波形生成给了系统最大的声音塑形自由度，不受固定合成拓扑限制
- 传统合成 fallback 确保底线——声音必须好听，这是不可妥协的硬约束
- 两种路径的切换点留待实现阶段决定（是 per-voice 切换还是全局切换）

### 2.6 为什么每 Voice 独立发育？

**决策**：5 个 Voice，每个是独立的单音合成器，拥有独立的发育史和"人格"。

**淘汰的方案（共享发育）**：所有 Voice 从中读取统一人格。淘汰原因：与"培育多个生命体"的隐喻冲突。如果全部共享，演奏者无法针对不同音高范围进行差异化培育。

**讨论要点**：
- Voice 身份由其在机器上的物理槽位定义（Voice 1-5），不绑音高
- 每次弹奏分配到哪个 Voice 由 voice allocation 策略决定（待定：循环 / 最近最少用 / 音高亲和度）
- 演奏者的心理模型："我有 5 个声音伙伴，每次按下一个键，其中一位在回应我"

### 2.7 为什么 Voice 间关系以竞争为主？

**决策**：Voice 间关系以频谱资源竞争为主导，能量串扰和共鸣耦合为辅。

**讨论要点**：
- 竞争的必要性：5 个独立发育的 Voice 共享可听频谱空间，若没有智能调度，要么整体频谱过满听感混乱，要么某个"肥胖"Voice 挤占所有空间
- 竞争机制不是传统压缩器——是一个有记忆的、神经网络驱动的频谱调度器：
  - 实时感知每个 Voice 占用的频谱区域和能量密度
  - 基于积累历史决定资源分配（而非瞬时音量）——经历过数百小时高"密"能量培育的 Voice，在低频段有更高的话语权
  - 竞争是软的、有性格的——不同能量方向培育出的 Voice，"退让风格"不同（有的让出谐波、有的让出瞬态前沿、有的让出尾音）
  - 结果：频谱永远不会"满"，但空出的方式有表情
- 串扰/耦合的作用远小于竞争：当一个 Voice 被注入能量时，微弱泄漏给相邻 Voice（谐波关系越近泄漏越多）；长期下来，频率/相位关系密切的 Voice 对可能形成耦合路径

## 3. 能量方向系统

### 3.1 四个能量方向

#### 张 (Tension) — 谐波锁紧

- **瞬时反馈**：相邻分音的频率比向整数比靠拢。声音从"散"（略带非谐波偏移）变"实"（谐波系列收紧）。频谱峰变得更锐利、更集中。
- **短期行为**：注入"张"能量后，声音听起来更"准"、更"集中"，谐波对齐带来的拍频减少。
- **中期行为**：反复注入"张"，系统发展出更尖锐的共振。相邻分音之间的锁紧力增强，系统需要更大的扰动才能从锁定状态脱离。
- **长期发育**：系统进入"高张力相"——默认状态谐波高度锁定，即使不注入"张"，声音也是结实、集中的。但在这种状态下，注入"扰"的瞬时效果会更剧烈（因为结构更脆、反作用力更大）。

#### 扰 (Turbulence) — 边带分叉

- **瞬时反馈**：当前活跃的每个谐波峰的旁侧分裂出一对非谐波小峰。声音从"平滑"表面裂出纹理——像一根弦在振动中突然多了一丝边缘的沙沙声。
- **短期行为**：边带峰短暂出现后逐渐衰减。频谱从"干净"变成"有纹理"。
- **中期行为**：反复注入"扰"，边带分叉路径被强化。不注入"扰"时，边带也在低水平持续存在——声音默认带上了一种"毛边质感"。
- **长期发育**：系统进入"分叉相"——声音默认状态下每个谐波都带稳定的旁瓣结构。注入"张"的瞬时效果变得更显著（收紧一个已经分叉的结构，产生复杂的中频运动）。

#### 密 (Density) — 路径增生

- **瞬时反馈**：低频至中低频段激活更多平行共振路径。声音从"薄片"变"实心"——频谱质量感增加，低中频变得有重量。
- **短期行为**：新增路径提供更多人耳可感知的"体积感"。声音不再是从一个点发出的单薄信号，而是有内部密度的实体。
- **中期行为**：反复注入"密"，增生路径被强化并向上扩散到中频和高中频。声音整体变得更"厚重"——不是更响，是更"多"。
- **长期发育**：系统进入"增生相"——默认状态下声音丰满、密集。但在这种状态下，Voice 在频谱竞争中占据更大空间，对其他 Voice 的挤压更强，Voice 间竞争关系需要更活跃的调度。

#### 忆 (Memory) — 状态回声

- **瞬时反馈**：系统过去被强激活过的连接模式以低音量混入当前声音输出。听起来像是在当前声音后面跟了一条"影子"——不是混响（不衰减），不是延迟（不重复），而是类似的声音"质感的记忆"被叠加上来。
- **短期行为**：回声持续存在但不衰减——更像是声音的"光谱里多了一个过去层"。
- **中期行为**：反复注入"忆"，系统的历史路径变得更易被重新激活——旧的状态不再是"被存档"，而是"随时可能渗入当前"。
- **长期发育**：系统进入"回声相"——默认状态下声音自带多层次的历史质感。演奏者弹一个音，听到的是"现在的音 + 过去几个月的痕迹叠加"。注入大量"忆"可能导致 Voice 陷入"记忆漩涡"——过去的模式越来越主导当前输出，新鲜输入被淹没（此时需要"张"或"扰"来打破）。

### 3.2 能量注入的操作接口

- **按键**：Gate/Trigger 类型。按下即注入，松开即停。力度/触后作为注入强度的调制源。
- **Patch Point**：Audio rate 输入。外部信号（其他 Voice 输出、外部音频、调制源）可直接作为"能量载体"注入。信号的特征（频谱、包络、瞬态）影响注入效果。
- **方向间关系**：纯正交——四个方向独立操作，组合注入产生涌现交互。任何互相抑制/增强都是系统自身动力学的结果，不是预设的语义关系（如"张抑制扰"之类的硬规则不存在）。

### 3.3 相变阈值

每个能量方向在其累积路径被强化到一定程度后，触发**相变**——系统在该维度上的默认状态发生不可逆的定性改变。相变是设计目标，不是 bug：

- 相变前：注入能量 → 瞬时反馈 → 逐渐衰减回默认状态
- 相变后：默认状态本身被改变 → 不再需要注入就能维持新的行为模式
- 相变不可逆——这是"发育"与"调制"的根本区别

相变的具体阈值和各方向的相互作用是实验性参数，需在实际系统中试出。

## 4. Voice 架构

### 4.1 Voice 身份

- 5 个独立 Voice，在物理槽位（Voice 1-5）上定义身份
- 每个 Voice 拥有独立的：
  - 神经网络音频生成模型（权重即拓扑）
  - 四方向能量积累状态（张/扰/密/忆的历史累计值）
  - 发育阶段（相变状态）
  - 竞争权重（在频谱调度中的话语权）
- Voice 身份不绑音高。Voice allocation 策略待定（循环 / LRU / 音高亲和度）

### 4.2 单 Voice 内部结构

```
┌──────────────────────────────────────────────────┐
│                   Voice N                         │
│                                                   │
│  Energy Inputs (4 × audio rate)                   │
│   张 ──→ [Energy Router] ──→ [Topology Modulator]│
│   扰 ──→                │                        │
│   密 ──→                │      ┌──────────┐      │
│   忆 ──→                └──→───┤ Neural   ├──→ Out│
│                                │ Audio    │      │
│  Pitch/Gate ──→ [Exciter] ──→ │ Generator│      │
│                                └──────────┘      │
│  State Memory (FRAM)                             │
│   ├─ Weight snapshot                             │
│   ├─ Energy accumulation history                 │
│   ├─ Phase state                                 │
│   └─ Competition profile                         │
└──────────────────────────────────────────────────┘
```

- **Exciter**：将音高和 gate 信息转化为神经音频生成器的输入激励
- **Energy Router**：将四路 energy input 路由到拓扑调制器的对应路径类型
- **Topology Modulator**：根据能量输入，实时调整神经网络的连接权重（短期）并累积结构变化（长期）
- **Neural Audio Generator**：核心音频生成，直接输出波形
- **State Memory**：FRAM 持久化存储

### 4.3 Fallback 路径

若 Neural Audio Generator 的声音质量不满足要求：
- 切换到传统合成结构（FM / 物理建模 / 波表 + 动态路由）
- Topology Modulator 的权重变化映射到传统合成参数的变化
- 切换粒度待定（per-voice 独立切换 / 全局切换 / 混合——部分 Voice neural、部分传统）

## 5. Voice 间动力学

### 5.1 频谱资源竞争

核心机制：**神经网络驱动的记忆型多段频谱调度器**

- 感知层：实时分析 5 个 Voice 的瞬时频谱占用（频段 × 能量密度）
- 决策层：基于每个 Voice 的竞争权重决定资源分配
- 执行层：对每个 Voice 施加软的、有性格的频谱塑形（非硬切）

竞争权重的决定因素：
- Voice 的能量积累历史（高"密"能量培育 → 低频段话语权增大）
- Voice 的发育阶段（经历多次相变的 Voice 获得更稳定的频谱"地盘"）
- 瞬时频谱重叠程度（重叠越多，调度越活跃）

退让风格（不同培育方向的 Voice 在冲突时的让步方式）：
- 张主导 Voice：让出瞬态前沿，保留稳态谐波
- 扰主导 Voice：让出谐波区，保留旁瓣纹理区
- 密主导 Voice：让出高频，保留低频段
- 忆主导 Voice：让出当前信号，保留回声层

### 5.2 能量串扰

- 注入一个 Voice 的能量，微弱泄漏给其他 Voice
- 泄漏量与频率距离相关（谐波关系越近，泄漏越多）
- 泄漏方向性：高竞争权重的 Voice 泄漏给低权重的多，反之少

### 5.3 共鸣耦合

- 长期共同演奏的 Voice 对（频繁同时激活的 pair），在频率/相位关系上形成耦合路径
- 耦合足够强时，弹一个 Voice 可能自动激发另一个——不是硬触发，是软共鸣
- 耦合可以被"张"能量锁紧、被"扰"能量松开

## 6. 多时间尺度发育模型

| 尺度 | 时间 | 现象 | 可感知变化 |
|------|------|------|-----------|
| **短期** | 毫秒-秒 | 能量注入的瞬时路由与声音响应 | 按下"扰"，边带分叉出现；松开后逐渐消退 |
| **中期** | 秒-分钟 | 方向间竞争/协同，系统在相空间里的轨迹 | 反复注入"密"，Voice 变得更厚重；注入"张"收紧结构 |
| **长期** | 月-年 | 系统结构的持久改变，相变后的新基线 | 数百小时演出后，Voice 的默认状态已与出厂时完全不同；某些方向更敏感、某些路径更"熟悉" |

**跨 Session 连续性**：每次演奏结束时的 Voice 状态（权重快照 + 能量积累 + 竞争配置）存入 FRAM。下次开机时从 FRAM 恢复——Voice 的发育跨越 session。

## 7. 记忆系统

### 7.1 存储

- 存储介质：FRAM（铁电存储器），非易失、可多次写入、无磨损问题
- 每个 Voice 存储内容：
  - 神经网络权重快照（当前拓扑状态）
  - 四方向能量积累历史值（张/扰/密/忆的累积量）
  - 相变状态记录（经历了哪些相变、当前处于哪个相）
  - 竞争配置（频谱权重、退让风格参数）
  - Voice 间耦合状态（与其他 Voice 的耦合强度和相位关系）

### 7.2 导出 / 遗传

- 单个 Voice 的完整状态可导出为文件
- 可导入到另一台机器上的某个 Voice 槽位——"遗传"
- 导出格式：二进制（权重数据 + 元信息 JSON）
- 导出内容可选：完整快照 / 仅权重 / 仅能量历史 / 仅竞争配置

### 7.3 快捷键唤起

- 快捷键可调出过去的某个记忆快照
- 两种唤起模式：
  - **跳转模式**：当前 Voice 状态替换为历史快照（"回到那个状态"）——破坏性操作，需确认
  - **注入模式**：历史快照以类似"忆"能量的方式注入当前状态，渐进混合——非破坏性，产生"过去的自己影响了现在的自己"的效果
- 快照自动创建触发条件待定（每次 session 结束 / 每次相变 / 手动标记）

## 9. 技术架构方向

### 9.1 神经网络选型方向

- 候选：RAVE（实时音频变分自编码器）、DDSP（可微分数字信号处理）、SNN（脉冲神经网络，适合时间动力学）、RNN/GRU 变体
- 评估维度：实时性（推理延迟 < 10ms）、可训练性、拓扑可调制性、声音质量
- **选型已完成** → [NEURAL_ARCHITECTURE_SURVEY.md](./NEURAL_ARCHITECTURE_SURVEY.md)
- **首选方案**：DDSP + Hypernetwork — Encoder 权重调制（长期发育）+ DSP 参数偏置（短期反馈）
- **长期储备**：SNN 混合架构（STDP + 神经调节），待神经形态硬件和 SNN 音频生成研究成熟

### 9.2 平台

- 首选：带硬件 AI 加速器的嵌入式平台（如含 NPU 的 SoC），满足实时推理 + FRAM 接口
- 原型阶段可在桌面端（Python/C++ + GPU）验证核心算法

### 9.3 核心挑战

- **实时拓扑调制**：在推理过程中动态改变神经网络权重，同时保持声音连续性
- **相变检测**：如何定义和检测能量积累是否达到相变阈值
- **频谱调度器训练**：如何训练一个有记忆的、有个性的频谱竞争调度器
- **Voice 间关系的涌现**：串扰和耦合如何在长时间运行中自然形成
- **训练数据**：神经音频生成模型的训练数据策略（合成数据 / 录制数据 / 在线学习）

### 9.4 Phase 1 完成 — DDSP 基础验证 (2026-05-11)

**模型架构**：
- `DDSPDecoder`: MLP(2→180) → GRU(180→180) → MLP(180→180) → harm_head(100) + noise_head(65)
- 258,465 参数（仅 decoder，harmonic/noise synth 无学习参数）
- 无 learned encoder（Phase 1 用固定 DSP 提取 f0/loudness 作为输入）
- `WavetableHarmonicSynth`：IFFT 构建单周期波形 → 查表 + 线性插值合成
- `FilteredNoiseSynth`：白噪声 → FFT → mel filterbank → IFFT

**训练数据**：135 个合成长笛片段（E2-C6, 3 力度层, 9 分钟）

**训练结果**（3000 steps, MPS GPU）：
- val_loss: 10.22 → 3.79 (-62.9%)

**延迟基准**（CPU）：
- 每帧 0.192ms（Decoder 0.082ms, Harmonic 0.093ms, Noise 0.017ms）
- 实时因子 **0.048**（20× 快于实时，远超 <10ms 硬约束）
- Wavetable 优化是关键——若无此优化，harmonic synth 将占主导成本

**代码位置**：`synth/` (core), `scripts/` (entry points), `configs/phase1.yaml`, `tests/`

### 9.5 Phase 2 完成 — 能量→参数映射 (2026-05-11)

**实现方式**：手动设计的信号处理规则，无 hypernetwork。四个方向独立可组合。

**EnergyBiasModule** (`synth/energy/biases.py`) — 无学习参数，纯信号处理：

| 方向 | 机制 | 状态 |
|------|------|------|
| 张 (Tension) | 幂律变换谐波锐化：harm^γ, γ=1+level×4 | 静态 |
| 扰 (Turbulence) | 每谐波 AM 调制(25-55Hz)产生音频速率边带 + 噪声 mel 波纹增强(×1.2) | 动态（AM 相位 buffer）|
| 吟 (Resonance) | 频谱倾斜扫频(0.25-1.2Hz, ±80% depth)，dark↔bright 连续摆动，反馈调制扫速 | 动态（时钟相位+反馈 buffer）|
| 忆 (Memory) | 600ms 延迟线环形缓冲快照混合（配合音高变化可听）| 动态（延迟线 buffer）|

**交互式测试** (`scripts/phase2_interactive.py`)：
- 实时 DDSP 推理（单帧 GRU + energy bias + DSP 合成）
- 音频 callback 线程 + curses TUI（键盘 1-4 toggle 能量方向, ↑↓←→ 调音高/响度）
- 能量有 attack (30ms) / release (150ms) 平滑防止 click
- `--render` 模式支持离线渲染到 WAV
- 依赖：`sounddevice`

**监听测试** (`outputs/phase2/`)：
- `baseline.wav` — 原始音色
- `tension.wav` — 谐波收紧效果
- `turbulence.wav` — 边带纹理效果
- `resonance.wav` — 共振流动效果
- `memory.wav` — 历史回声效果
- `phase2_demo.wav` — 四方向依次 ramp

**量化验证**（100ms chunk RMS 分析）：
- 张：RMS -47%，peak -65%，强弱谐波比 6000×
- 扰：peak +30%，RMS 不变，AM 边带纹理
- 吟：RMS max/min = **4.5×**（基线仅 1.3×），~1.4s dark↔bright 扫频周期

**A/B 监听文件**：`outputs/phase2/ab/{tension,turbulence,resonance,memory}_ab.wav`

### 9.6 诊断 — 音色可调范围窄的根因 (2026-05-11)

Phase 1+2 完成后，实际演奏测试发现声音可调范围非常窄。这不是参数调试问题，是**结构性瓶颈**，有三层根因：

**第一层：训练数据只有一种音色。** 135 个片段全部来自 `scripts/generate_data.py` 中的单一长笛谐波模型——同一个 `1/h` 衰减 + `even_odd_ratio=0.7` 公式生成。Decoder 学到的 `(f0, loudness) → 谐波结构` 映射被困在一个"长笛流形"上。无论输入什么 f0/loudness，输出永远是长笛。

**第二层：Decoder 输入空间只有 2 维。** 当前架构跳过了 learned encoder，输入只有 `[f0_scaled, loudness]` 两个标量。标准 DDSP autoencoder 有 `encoder → z → decoder`，z 是一个高维 latent 向量可以编码丰富的音色信息。2 维输入意味着模型根本没有表达"不同音色"的通道——f0 和 loudness 的变化只能改变音高和响度，不能改变音色本体。

**第三层：能量偏置是后处理，不是生成。** Phase 2 的四个能量方向（张/扰/吟/忆）全部在 decoder 输出上做确定性信号处理。它们可以锐化、调制、扫频、混合一个已有的谐波结构，但不能生成一个不同性质的谐波结构。后处理的原材料永远是一个长笛。

**为什么后续的 hypernetwork 阶段不能自动解决这个问题：** Hypernetwork 做 `energy → ΔW`，在权重空间中导航。但如果 decoder 只在长笛数据上训练过，它的权重空间就是一个"长笛流形"——hypernetwork 走到哪，产出的都是长笛的变形。要让 hypernetwork 的导航有足够大的疆域，**decoder 需要先在多样化的音色数据上训练**。这是 hypernetwork 阶段的前提条件，不是它能绕过去的。

**对训练数据策略的影响：** 开放问题 #7（训练数据策略）需要从"合成数据可行"更新为"需要多音色训练数据，覆盖不同乐器类别、不同谐波结构模型"。这是下一阶段工作的前置依赖。

### 9.7 Phase 3 完成 — 多 Voice 独立发育与频谱竞争 (2026-05-11)

**架构**：1 个共享 DDSPDecoder + 5 个独立 VoiceModule + SpectralCompetitionScheduler + VoiceAllocator (round-robin)

**VoiceModule** (`synth/voice.py`)：
- 每个 Voice 持有独立 GRU hidden state、EnergyBiasModule 实例、VoiceState
- VoiceState 包含：四方向能量积累（累计秒数）、不可逆相变标记、竞争权重、退让风格（3 频段增益因子）
- 两步推理接口：`process_params()` → competition → `synthesize_from()`
- 相变后自动叠加 phase baseline（floor=0.08）到对应方向

**能量积累**：
- 公式：`accum[direction] += effective_level * frame_duration`（frame_duration = 4ms）
- 能量注入与音符触发解耦——Voice 静音时也积累能量
- 相变阈值：每方向 5.0s 累积（level=1.0 等效）

**竞争权重推导**：`weight = 1.0 + total_accum * 0.1 + phase_count * 0.3`

**退让风格**：由四方向积累比例混合计算，张主导→让出中频少（0.30），扰主导→让出中频多（0.70），吟主导→让出高频多（0.70）、让出低频少（0.30），忆主导→均匀（0.50）

**SpectralCompetitionScheduler** (`synth/competition.py`)：
- 最简可用版本：在谐波振幅域（非音频域）调度，避免 FFT 开销和 artifacts
- 3 频段分界：low <500Hz, mid 500-2000Hz, high >2000Hz
- 每频段总能量超阈值（0.5）时，按竞争权重分配资源
- 超额的 Voice 按退让风格承受差异化软增益衰减（最低 0.3×）

**Voice Allocation**：循环分配（round-robin），不绑音高。新音符总是分配给下一个 Voice。

**离线渲染验证** (`scripts/phase3_render_demos.py`)：
- `independent_voices.wav` — 5 Voice 各自不同能量方向培育，36s，验证发育差异化
- `spectral_competition.wav` — 3 Voice 同音高竞争 + 八度分散对比 + 能量注入竞争，18s
- `phase_transition.wav` — 单 Voice 持续注入张能量→相变，12s
- 确定性验证：torch.manual_seed(42) 后逐位相同

**交互式测试** (`scripts/phase3_interactive.py`)：
- 多 Voice 选择（1-5），每 Voice 独立能量注入（qwer），音符触发（Space），音高/响度调节
- TUI 显示 5 Voice 状态矩阵：能量积累、相变标记、竞争权重、退让风格、活跃音符

**代码位置**：`synth/voice.py`, `synth/competition.py`, `synth/poly.py`, `scripts/phase3_*.py`

### 9.8 Phase 4 完成 — 多音色训练数据扩展与 Decoder 重训练 (2026-05-11)

**根因回顾** (§9.6)：decoder 只在单一长笛音色上训练，权重空间被困在"长笛流形"上。Phase 4 通过生成 4 类不同谐波结构的合成数据并重训练 decoder，扩展了权重空间的基础疆域。

**数据生成** (`scripts/generate_multitimbral_data.py`)：
- 4 类音色：弦乐 (string)、铜管 (brass)、人声 (voice)、电子 (electronic)
- 每类 135 片段 (MIDI 40-84 × 3 力度层)，总计 540 片段
- 类内参数随机化覆盖"中间态"：harmonic decay rate, even/odd ratio, formant positions, inharmonicity β 等
- 确定性：`base_seed=42` → 逐位相同输出

**音色模型特征**：
| 类 | 谐波特征 | 包络 | 噪声 | 关键随机化参数 |
|----|---------|------|------|--------------|
| string | 1/h^α (0.5<α<0.9), 偶次增强 1.0-1.8× | mod attack, 高 sustain | 弓噪声 (2kHz LP) | decay, even_ratio, bow_level |
| brass | 1/h^α (0.8<α<1.2), 奇次主导 0.2-0.5× | fast attack 10-40ms | 气息噪声 (1kHz HP) | even_ratio, brightness_coupling |
| voice | 1/h + 共振峰包络 + 元音切换 | mod attack, formant-dependent | 气声噪声 (级联 LP+HP) | formant_freqs, jitter, vowel_pair |
| electronic | 1/h^α (0.3<α<1.5) + 非谐波偏移 β | sharp attack 0-5ms, 可变 decay | 白/粉噪声 | β, sub_bass_boost, spectral_notch |

**训练结果** (3,000 步, MultiScaleSpectralLoss)：
- 混合数据 val_loss: **5.97** (Phase 1 纯长笛: 3.79)
- 分类 val_loss: string=4.63, voice=5.31, brass=5.53, electronic=7.87
- 损失升高 = 权重空间张力 = 超网络导航空间。若混合数据仍能完美拟合，超网络将无事可做

**关键洞察**：
- 电子类最高 loss (7.87)：非谐波频谱与纯谐波合成后端的结构性冲突——这是超网络 ΔW 能最大程度发挥的"杠杆点"
- 弦乐类最低 loss (4.63)：与原始长笛谐波结构最接近，模型的流形在此区域最"熟练"
- 2D 输入瓶颈依然存在：decoder 无法为不同音色产生不同输出，但已在更多样化的谐波结构上训练过——后续 hypernetwork 的 ΔW 导航有足够广阔的权重空间可探索

**代码位置**：`scripts/generate_multitimbral_data.py`, `data/multitimbral_raw/`, `data/multitimbral_processed/`

## 附录：设计讨论的过程记录

### 初始方向探索

用户提出了"在不同正交方向上注入能量，拥有短中长期可感知变化"的核心概念。经过讨论，明确了：

- 声音生成方式保持开放，神经音频合成优先，传统合成 fallback
- 能量方向为显式操作——演奏者有意识地选择往哪个方向培育
- 方向语义选择"能量类型"（演奏者投注不同性质的"能量"），而非"生长维度"
- 能量注入的物理对应选择"拓扑注入"（改变系统内部连接路径）

### 能量方案选择

对比了两个方案：
- 方案 A（激/润/裂）：按"能量作用于声音的哪个维度"分类，短期感知清晰，长期发育因果链断裂
- 方案 B（张/扰/密/忆）：按"能量对系统动力学轨迹的影响"分类，更适合长线条发育

选择方案 B，但指出原始描述过于宽泛（"扰"按下后听到什么？），每个方向被细化到具体的、精确的感知签名。

### Voice 间关系

确认以频谱资源竞争为主导机制——5 个 Voice 共享可听频谱空间。竞争由神经网络驱动的、有记忆的频谱调度器管理。能量串扰和共鸣耦合作为次要的涌现机制存在。

### 记忆系统

明确使用 FRAM 存储，支持导出、遗传（跨机器迁移）、快捷键唤起（跳转/注入两种模式）。
